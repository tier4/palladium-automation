## GitHub統合方式の概要

### 旧方式（非推奨）

1. **ローカル**: スクリプトをxdotoolでETXに転送
2. **ETX**: バックグラウンドで実行
3. **ETX**: 結果をGitHubにpush
4. **ローカル**: GitHubをポーリング（10秒間隔）
5. **ローカル**: 結果取得後、GitHubから削除

**問題点**:
- 結果回収に10-30秒かかる
- GitHubへの依存
- 複雑な実装（350行）
- xdotoolの不安定性

### 現方式（推奨）

**SSH同期実行**:
```bash
ssh ga53pd01 "bash -s" < script.sh 2>&1 | tee result.txt
```

**利点**:
- 高速（2-3秒）
- シンプル（130行）
- リアルタイム出力
- 外部サービス不要

## パフォーマンス比較

| 項目 | GitHub方式 | SSH同期実行 |
|------|-----------|------------|
| 実行時間 | 10-30秒 | 2-3秒 |
| 実装行数 | 350行 | 130行 |
| 外部依存 | GitHub | なし |
| リアルタイム出力 | ❌ | ✅ |
| 安定性 | 中 | 高 |
